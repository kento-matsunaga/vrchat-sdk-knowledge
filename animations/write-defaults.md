# Write Defaults（WD）

VRChat SDKで最も混乱を招く設定の一つ。アバター全体で統一しないと不具合が発生する。

---

## Write Defaults とは

AnimatorのステートにあるMotionの設定。  
**そのステートがアニメーションしていないプロパティを「デフォルト値に戻すか」を制御する。**

| 設定 | 動作 |
|------|------|
| **ON（デフォルト）** | ステートがアクティブな間、アニメーションに含まれないプロパティはAnimatorのデフォルト値（アバター初期状態）に戻る |
| **OFF** | ステートがアクティブな間、他のステートやLayerが変更したプロパティはそのまま保持される |

---

## なぜ混在させてはいけないか

**WD ONとWD OFFを混在させると:**
- あるステートに入った時に意図しないプロパティがリセットされる
- 表情が中途半端に残ったり、消えたりする
- 特にFXレイヤーとGestureレイヤーが干渉して不具合が出やすい

**VRChat公式の推奨:**
> "アバター全体でWrite Defaultsの使用を一貫させてください。全てOFFまたは全てONで統一してください。"

---

## WD OFF推奨の理由（コミュニティでの主流）

現在のVRChatアバター制作コミュニティでは **WD OFF** が主流。

**理由:**
- WD ONは「初期状態に戻る」という挙動が直感に反することが多い
- 複数のLayerを組み合わせた複雑なアバターでWD ONは制御が難しい
- WD OFFなら各ステートが明示的に制御したプロパティのみ変更される

---

## WD OFF を使う際のルール

WD OFFの場合、**全ての状態を明示的にアニメーションする**必要がある。

### 例: ObjectのON/OFFトグル

**間違い（WD OFF時）:**
```
State "Item ON"  → アニメーション: GameObject.Active = true のみ
State "Item OFF" → アニメーション: なし
→ Item OFFステートに入ってもGameObjectがONのまま残る
```

**正しい（WD OFF時）:**
```
State "Item ON"  → アニメーション: GameObject.Active = true
State "Item OFF" → アニメーション: GameObject.Active = false
→ 両方のステートで明示的に設定する
```

---

## 確認・修正方法

### 一括確認（推奨ツール）
**Av3 Creator / VRCFury / Modular Avatar** などのツールを使うと  
WDの整合性チェック・一括修正が可能。

### 手動確認
Animatorウィンドウで各ステートを選択 → InspectorでWrite Defaultsのチェックボックスを確認

---

## まとめ

| ケース | 推奨設定 |
|-------|---------|
| 新規作成するアバター | WD OFF（全ステートで統一） |
| 既存のアバターに追加 | 既存アバターに合わせる（混在させない） |
| デフォルトのVRChatコントローラー使用 | WD ONのものが多い → 追加部分もONで統一 |
